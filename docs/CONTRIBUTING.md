# Guia de Contribuição

Agradecemos o seu interesse em contribuir para o projeto aBible!

## Padrões de Código

-   **Formatação**: Use o `dart format` para formatar o código.
-   **Análise Estática**: Siga as regras definidas em `analysis_options.yaml`.
-   **Nomenclatura**: Siga as convenções de nomenclatura do Dart/Flutter.

## Fluxo de Trabalho de Desenvolvimento

1.  **Fork o Repositório**: Crie um fork do projeto para a sua conta do GitHub.
2.  **Crie uma Branch**: Crie uma branch para a sua feature ou correção (`git checkout -b feature/sua-feature`).
3.  **Faça Commits Claros**: Escreva mensagens de commit claras e concisas.
4.  **Abra um Pull Request**: Envie um Pull Request para a branch `main` do repositório original.

## Submissão de Alterações

-   Certifique-se de que o seu código compila e executa sem erros.
-   Se aplicável, atualize a documentação relevante.
-   Descreva as suas alterações em detalhes no Pull Request.
